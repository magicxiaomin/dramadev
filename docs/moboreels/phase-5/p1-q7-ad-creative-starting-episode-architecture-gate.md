## Architecture summary

The Q7 decision packet preserves the existing SceneFlow conversion architecture: Facebook ad click lands directly on Watch EP1, the user watches the ordered free episode chain, reaches the first locked episode, sees the Unlock Drawer, and fake unlock/pass returns to that same locked episode with `unlocked=1`.

The proposal is intentionally conservative. EP1 is the PRD-defined default route, the first named funnel event after Watch Page Loaded, the QA checklist's starting point, and the least risky way to preserve narrative context before the first lock. Non-EP1 starts can be valuable later, but only as opt-in fake-only experiments constrained to same-story free episodes and never as default traffic behavior.

Prototype B remains only a secondary pointer to the SceneFlow PRD; it adds no competing implementation requirement.

## P0 invariant preservation

Invariant, quoted exactly:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook -> free episode chain -> first locked episode -> Unlock Drawer -> fake unlock/pass -> same episode with unlocked=1
```

Q7 preserves this invariant if and only if these guardrails remain true:

- Default Facebook-ad traffic starts at EP1, not Home, Search, Browse/Genre, Show Detail, Pass, login, recharge, payment, install education, or a creative-specific mapped episode.
- `episode=1` remains the default start parameter for P0/default P1 Facebook traffic.
- `source=facebook` and any future safe attribution params are inert route context only unless separately approved; they must not alter playback, lock detection, or same-episode return.
- Free episodes remain watchable before login, recharge, Story Pass promotion, PWA install prompt, payment-like prompt, real analytics, Facebook processing, backend lookup, or entitlement check.
- The free chain proceeds in episode order until `episode > story.freeEpisodes && unlocked !== true`.
- The first locked episode opens the Unlock Drawer; closing the drawer keeps the user on the locked episode; tapping locked playback reopens the drawer.
- First-lock CTA hierarchy remains primary single-episode unlock and secondary Story Pass.
- Fake single-episode unlock and fake Story Pass return to `/variant-b/watch/[showId]?episode=[lockedEpisode]&unlocked=1`, preserving the show and episode actually reached from the free-chain path.
- `unlocked=1` remains fake review state only and never production entitlement.

## Route and query implications

Current safe default route remains:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook
```

Allowed route/query semantics for this gate:

- `episode=1` is the default start episode for Facebook-ad traffic.
- `source=facebook` may be preserved through watch/pass/show-detail navigation as safe attribution context.
- Existing optional attribution params may be preserved when present, but must remain inert for Q7.
- Fake unlock return remains `/variant-b/watch/[showId]?episode=[lockedEpisode]&unlocked=1`.
- Fake Story Pass may use `/variant-b/pass?story=[showId]&episode=[lockedEpisode]` and must return to the same watch episode with `unlocked=1`.

Rejected route/query behavior:

- Defaulting a Facebook-ad click to `episode>1`.
- Allowing `creative_id`, `utm_content`, or any ad/campaign value to select a starting episode by default.
- Falling back to Home/Search/Show Detail/Pass for invalid mapping.
- Starting at a locked episode, after the first locked episode, outside the same story, outside the free range, or in a way that shortens/skips the free episode chain.
- Dropping `showId`, `episode`, `source`, safe attribution, or same-episode return context.
