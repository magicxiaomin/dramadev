## P0 invariant preservation

Invariant, quoted exactly:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook -> free episode chain -> first locked episode -> Unlock Drawer -> fake unlock/pass -> same episode with unlocked=1
```

No-regression constraints:

- Facebook-ad users continue to land directly on `/variant-b/watch/[showId]?episode=1&source=facebook`, not Home, Search, Browse, Genre, Show Detail, Pass, login, recharge, or Story Pass education first.
- Free preview remains prompt-free: no login, recharge, Story Pass prompt, subscription prompt, PWA install prompt, homepage redirect, detail redirect, payment-like CTA, or real-service dependency before the first locked episode.
- The first locked episode remains `freeEpisodes + 1` unless explicitly unlocked by the fake URL flag.
- The first locked episode opens the Unlock Drawer; drawer close leaves the user on the same locked episode; tapping locked playback reopens the drawer.
- The first-lock drawer CTA hierarchy remains primary single-episode unlock (`Unlock EP X`, or fake insufficient-balance `Get coins to unlock` if that branch is separately exercised), secondary `Get Story Pass`, tertiary `Maybe later`.
- Fake single-episode unlock returns to `/variant-b/watch/[showId]?episode=[episodeNumber]&unlocked=1`, preserving safe attribution.
- Fake Story Pass routes through `/variant-b/pass?story=[showId]&episode=[episodeNumber]` and returns to the same watch episode with `unlocked=1`; it must not return Home, strand users on the Pass page, or lose `episode` context.
- `unlocked=1` remains fake review state only; it is not production entitlement, subscription, login, account, payment, backend, database, or wallet state.
