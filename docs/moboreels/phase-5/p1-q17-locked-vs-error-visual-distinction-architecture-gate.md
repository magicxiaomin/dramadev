# Phase 5 P1 Q17 Architecture Scope Gate — Locked vs error visual distinction default

Status: ARCHITECTURE GATE APPROVE for docs-only / fake-only / non-implementation proposal material only. Product/design confirmation remains pending; architecture/video-provider confirmation remains required before any real playback or production error handling; runtime implementation remains separately gated.

## Final recommendation

Approve the Q17 decision packet as architecture-safe proposal material only: preserve a distinct fake-only locked-state vocabulary and interaction model that cannot be confused with video, weak-network, offline, provider, or playback error states.

The default is safe if locked states always read as intentional access gating and error states always read as technical recovery. Locked states should name the current episode and unlock action, use lock/unlock affordances, avoid fake playback progress, open or reopen the Unlock Drawer, and keep single-episode unlock as the primary first-lock CTA. Future video/network/offline error states should use retry/offline/error vocabulary and must never show unlock, Story Pass, coin, cost, payment, or entitlement CTAs.

P0 invariant, preserved verbatim:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook -> free episode chain -> first locked episode -> Unlock Drawer -> fake unlock/pass -> same episode with unlocked=1
```

## Architecture summary

Q17 is a taxonomy and scope gate, not a playback implementation. The PRD already requires the first locked episode state to be clear, not look like a network error, open the Unlock Drawer, remain locked when the drawer is closed, and reopen the drawer when the locked playback area is tapped. It also requires weak-network and offline states to remain distinct from locked state while real video remains deferred.

Therefore the safe architecture boundary is:

- Fake P0/P1 locked state is derived from local/story fixture state: `episode > story.freeEpisodes && unlocked !== true`.
- Locked state is a product/access state and may show `EP X is locked`, lock/unlock affordances, story hook, fake balance, fake cost, `Unlock EP X`, `Get Story Pass`, and same-episode return copy.
- Error state is a technical recovery state and may show retry/offline/weak-network/playback-unavailable copy once separately authorized.
- Error state must not open the Unlock Drawer or show unlock/pass/payment-like CTAs unless the user is separately on an actually locked episode under the per-drama lock rule.
- Distinction must rely on copy, icon/affordance, CTA labels, and behavior; color alone is not sufficient.

Current repo grounding confirms the existing stack remains within the expected prototype boundary: Next.js App Router, TypeScript, React, Tailwind, static fake data, client-side mock state, Vitest, and Playwright scripts. Current fake-only Watch surfaces already contain a locked boundary placeholder, `EP {episode} is locked` copy, no locked-playback progress, an `Open Unlock Drawer` action, drawer reopen behavior, fake balance/cost drawer copy, and mock same-episode `unlocked=1` return wiring. This gate does not require or authorize runtime changes.
