## Architecture summary

Q8 is safe only when optional attribution params remain passive context carried by URLs. The core SceneFlow architecture stays watch-first and free-preview-first: Facebook ad traffic lands directly on Watch EP1, continues through the ordered free episode chain, reaches the first locked episode, opens the Unlock Drawer, and fake unlock/pass returns to that same episode with `unlocked=1`.

The PRD already names the optional future attribution param set and explicitly says P0 does not need analytics processing and that those params must not disrupt routing or playback. Prototype B remains only a pointer to the SceneFlow PRD.

Current repo inspection shows the runtime already has fake-only helpers that parse/preserve a closed attribution key set including `source` plus the Q8 params. This task does not expand or rely on that runtime behavior; it records the architecture boundary for review.

## P0 invariant preservation

Invariant, quoted exactly:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook -> free episode chain -> first locked episode -> Unlock Drawer -> fake unlock/pass -> same episode with unlocked=1
```

Q8 preserves this invariant if and only if these guardrails remain true:

- Default Facebook-ad traffic still starts at `/variant-b/watch/[showId]?episode=1&source=facebook`, not Home, Search, Browse/Genre, Show Detail, Pass, login, recharge, payment, install education, or an attribution-derived route.
- `source=facebook` remains sufficient for P0 routing context; optional params are never required for playback or conversion flow success.
- Optional params must not change `showId`, `episode`, `unlocked`, `freeEpisodes`, lock state, drawer auto-open, CTA hierarchy, pass context, or same-episode fake unlock return.
- Free episodes remain playable before login, recharge, Story Pass promotion, PWA install prompt, payment-like prompt, real analytics, Facebook processing, backend lookup, cookies/storage, or entitlement checks.
- The free chain proceeds in episode order until `episode > story.freeEpisodes && unlocked !== true`.
- The first locked episode opens the Unlock Drawer; closing the drawer keeps the user on the locked episode; tapping locked playback reopens the drawer.
- First-lock CTA hierarchy remains primary single-episode unlock and secondary Story Pass.
- Fake single-episode unlock and fake Story Pass return to `/variant-b/watch/[showId]?episode=[lockedEpisode]&unlocked=1`, preserving the show and episode actually reached.
- `unlocked=1` remains fake review state only and never production entitlement.

## Route and query implications

Current safe default route remains:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook
```

Allowed route/query semantics for this gate:

- `episode=1` remains the P0/default P1 start episode for Facebook-ad traffic.
- `source=facebook` remains the P0 source marker.
- `campaign_id`, `adset_id`, `ad_id`, `creative_id`, `placement`, `utm_source`, `utm_campaign`, and `utm_content` may be preserved as inert URL context if present and if later implementation is separately authorized.
- Fake unlock return remains `/variant-b/watch/[showId]?episode=[lockedEpisode]&unlocked=1`.
- Fake Story Pass may use `/variant-b/pass?story=[showId]&episode=[lockedEpisode]` and must return to the same watch episode with `unlocked=1`.

Rejected route/query behavior:

- Using optional params to select a different show, starting episode, free-episode count, lock point, CTA hierarchy, drawer timing, pass return, or unlock state.
- Treating optional params as analytics events, attribution records, user identity, entitlement, personalization, or legal/consent state.
- Requiring optional params for P0 or failing the Watch flow when they are missing, duplicated, malformed, long, unknown, or extra.
- Adding `fbclid`, `utm_medium`, experiment IDs, or other non-PRD keys as default-approved params from this gate.
- Displaying optional params as user-facing trust, legal, pricing, monetization, or drawer copy beyond developer/debug metadata in fake-only review surfaces.
