## 4. Drawer presence requirements

At the first locked episode (`freeEpisodes + 1`) and at any later locked episode reached without `unlocked=1`, the Unlock Drawer must:

- Auto-open on first entry; drawer close leaves the page locked; tapping the locked playback area reopens the drawer.
- Render the mock balance inside the drawer at the stable test ID `data-testid="unlock-drawer-balance"`.
- Render the mock cost inside the drawer at the stable test ID `data-testid="unlock-drawer-cost"`.
- Show, alongside balance and cost: current story hook, drama title, locked episode number, primary `Unlock EP X` (or `Get coins to unlock` when `mockBalance < mockCostPerEpisode`), secondary `Get Story Pass`, tertiary `Maybe later` / close, and return-to-this-episode helper copy.
- Source balance and cost only from the show fixture (see §5). No runtime computation, network call, storage, cookie, or environment access for balance or cost.
- Keep CTA hierarchy unchanged: primary single-episode unlock (balance-conditional label, drawer-only), secondary Story Pass, tertiary close. Story Pass is never primary at first lock.
- After `mock-unlock-episode-link` or the Pass round-trip, return to the same episode with `unlocked=1` via `buildWatchEpisodeHref` / `buildPassReturnHref`, preserving `source=facebook` and any attribution params.

Balance and cost must not appear in any other surface: not in header, Watch page chrome, playback overlay, episode-complete CTA, Episode Sheet header, error/retry/offline states, or any pre-lock UI.
