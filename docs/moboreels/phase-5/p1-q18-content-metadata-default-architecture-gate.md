# Phase 5 P1 Q18 Architecture Scope Gate — fake-only content metadata default

Status: ARCHITECTURE GATE APPROVE for docs-only / fake-only / additive planning only. Product/editorial/content-rights confirmation remains pending.

## Final recommendation

Approve the Q18 decision packet as architecture-safe proposal material only: define a closed fake-only content metadata default for demo dramas, limited to the fields needed by the PRD Watch, Episode Sheet, first locked episode, Unlock Drawer, Story Pass/Pass return, and auxiliary Show Detail surfaces.

The recommended default is safe only if it remains a requirements/planning artifact until separately approved. It must not create runtime fixtures, schema validators, route loaders, tests, backend/database/content APIs, real video or asset pipelines, entitlement, payment/subscription/login, analytics/Facebook integrations, production infrastructure, legal/content decisions, or NovelHub production dependencies.

P0 invariant, preserved verbatim:

```txt
/variant-b/watch/[showId]?episode=1&source=facebook -> free episode chain -> first locked episode -> Unlock Drawer -> fake unlock/pass -> same episode with unlocked=1
```

## Architecture summary

Q18 can be accepted as a bounded fake-content planning contract because the PRD already requires drama title, episode number, tags, free/locked state, total episode count, free episode range, range tabs, current range, price/unlock condition, story hook, fake balance, fake cost, and same-episode return context. The metadata default should support those existing fake review surfaces without changing routing, playback, lock rules, CTA hierarchy, or unlock/pass return semantics.

The architecture boundary is a closed fake-only content contract, not a production data model. `freeEpisodes` remains the per-drama lock point; first locked episode remains derived as `freeEpisodes + 1` when `totalEpisodes > freeEpisodes`; `unlocked=1` remains fake review state only. Tags remain display labels, not analytics/ad-targeting/recommendation taxonomy. Episode ranges remain display metadata, not pagination/backfill/API behavior. Fake cost/balance remain drawer/pass copy, not pricing, wallet, entitlement, or payment rules.

Current repo inspection shows existing fake-only types and fixtures already contain related fields: `SceneFlowShow.id/title/logline/synopsis/genreTags/tropeTags/totalEpisodes/freeEpisodes/mockBalance/mockCostPerEpisode/firstLockedEpisode/episodeRanges/episodes` and `SceneFlowEpisode.episodeNumber/title/lockState/storyHook`. This gate does not authorize changing those fixtures or types; it records the scope boundary for any later implementation review.

## P0 invariant preservation

Q18 preserves the P0 invariant if and only if these guardrails stay true:

- Facebook ad traffic still lands directly on `/variant-b/watch/[showId]?episode=1&source=facebook`, not Home, Search, Browse/Genre, Show Detail, Pass, login, recharge, payment, install education, a CMS lookup, or an asset/content-selection step.
- EP1 remains the default P0/P1 ad-landing start episode unless a separately approved future ad-creative decision changes it.
- Free preview episodes play before login, recharge, Story Pass promotion, PWA install prompt, payment-like prompt, real attribution/content processing, backend lookup, or entitlement check.
- `freeEpisodes` remains the per-drama lock point source; metadata may not override lock state per episode except as derived fake display state.
- First locked episode remains `freeEpisodes + 1` when `totalEpisodes > freeEpisodes` and opens the Unlock Drawer when `unlocked !== true`.
- Closing the Unlock Drawer keeps the user on the locked episode; tapping the locked playback area can reopen the drawer.
- The first-lock CTA hierarchy remains primary single-episode unlock and secondary Story Pass.
- Fake single-episode unlock and fake Story Pass both return to `/variant-b/watch/[showId]?episode=[lockedEpisode]&unlocked=1` for the same show and episode.
- Metadata fields such as tags, range size, hooks, asset labels, cost copy, or content status cannot alter route behavior, starting episode, free chain order, drawer timing, CTA hierarchy, pass return, or fake `unlocked=1` semantics.
- `unlocked=1` remains fake-only review state and cannot become production entitlement.

## Metadata contract implications

Recommended drama-level fields for a later product-confirmed fake-only contract:

1. `showId` or equivalent stable fake identifier for `/variant-b/watch/[showId]`, pass, and detail links.
2. Fake `title` for Watch, Unlock Drawer, Episode Sheet, Pass, and Show Detail.
3. Safe short hook/logline and synopsis for Watch, Unlock Drawer, Pass context, and Show Detail.
4. Approved-safe genre/trope display tags.
5. `totalEpisodes` as a positive integer for Episode Sheet count and range generation.
6. `freeEpisodes` as the per-drama lock point source, with first lock derived from it.
7. Episode range display configuration, defaulting to 24 unless product/design confirms another value.
8. Required first-lock hook metadata for the first locked episode conversion moment.
9. Fake per-episode unlock cost and fake balance copy, clearly non-production.
10. Explicit asset-safety provenance/status flags for placeholder poster/thumbnail/video labels.
11. Content review status such as `fake-only-placeholder`, not production-ready.

Recommended episode-level fields for a later product-confirmed fake-only contract:

1. `episodeNumber` from 1 through `totalEpisodes`.
2. Safe fake `episodeTitle` or generated display label.
3. `isFree` / display free state derived from `episodeNumber <= freeEpisodes`.
4. `isFirstLocked` derived from `episodeNumber === freeEpisodes + 1` when a lock exists.
5. Locked-state copy distinct from video/network/error copy.
6. Required first-lock hook; later locked hooks optional/deferred unless separately approved.
7. Unlock condition copy for Episode Sheet, not just a lock icon.
8. Placeholder playback label indicating fake/demo content without implying real video infrastructure.

Rejected metadata behavior:

- Manually setting per-episode lock state that conflicts with `freeEpisodes` for the P0 path.
- Using tags, asset labels, campaign context, genre, title, or hook copy to choose start episode, lock point, CTA hierarchy, drawer timing, or return route.
- Requiring complete every-episode editorial metadata before P0/fake review can preserve the core free-chain path.
- Treating fake titles/hooks/posters/thumbnails/video labels as licensed, public, production, or third-party approved assets.
- Treating a metadata contract as a backend/database/API/CMS schema or NovelHub model.
